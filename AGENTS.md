# Управление задачами

Проект использует mdtask как основной источник задач и спецификаций.

## Обязательные правила

- Для создания новых задач используй skill `mdtask-add`.
- Для реализации существующих задач используй skill `mdtask-do`.
- Перед началом разработки запускай `mdtask list`.
- Получай содержимое задачи через `mdtask view <ID>`.
- Не изменяй статусы задач вручную, если это предусмотрено workflow mdtask.
- Не начинай реализацию функции, пока для неё нет задачи в mdtask.
- Выполняй только одну задачу за один цикл.
- После реализации обновляй спецификацию и блок `Implemented`.
- Одна завершённая задача должна соответствовать одному Git-коммиту.
- Если требуется решение пользователя, помечай задачу тегом `#user-required`.
